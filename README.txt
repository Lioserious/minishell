
    Commit messages
    "x = Affected area, e.g., parsing, i.e., FEAT(parsing): <Explanation of the commit>”
        ADD() —> new folder/file
        FEAT() —> Add new feature
        FIX() —> Bug fix
        NORM() —> Changes that do not affect meaning of code, for example Norm
        BUILD() —> Frame, architecture of Project
        CHORE() —> Boring changes, that had to be done
        REVERT() —> Revert a previous commit
    Example:
	
    FEAT(execution): Add execution frame for Buildins etc…
